TEACHING BEATS:
 (1) A mock proves your LOGIC; a real database proves your SQL, JPA mapping,
     constraints, @EntityGraph fetch, and type coercion actually work. The
     mocked ApprovalServiceTest cannot catch a broken @Query or a bad column
     map; the integration test can.
 (2) @ServiceConnection is the modern Spring Boot Testcontainers seam — no
     hand-wired JDBC URL; the container's connection details flow in automatically.
 (3) disabledWithoutDocker=true is the discipline that lets integration tests
     live in the same suite without blocking people who lack Docker — skip, don't
     fail. Proven live this session.
 (4) Honest limitation: the live 37/37 GREEN re-run needs Docker, which is down
     (machine state, documented). The skip-clean run IS real and complete; the
     all-green run is cited from the repo build. When the full stack is up this
     is a 2-minute re-confirm (same standing note as M-2.2).
