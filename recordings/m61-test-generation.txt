THE LESSON (all three patterns from one target):
  A generated test's value is what it would CATCH, not whether it is green.
  The PROMPT CONTRACT governs quality:
    - mirror the implementation -> certifies bugs (assert what the code DOES)
    - "make it pass" -> edits the code under test to satisfy tests (silently
      erased a planted bug here)
    - anchor to the SPEC + forbid source edits + "a failing test is the finding"
      -> catches the bug; RED is the deliverable.
  Naming: pattern-3 names are behavioral ("auto-approves a request scored exactly
  at the threshold"); garbage names mirror code ("returns true when score > 0.85").
  Foreshadows M-6.4 (mutation: a test only counts if it can fail) and M-6.3
  (BUG-WEB-03's untested form). BUG-WEB-04 left intact for its own reveal.
