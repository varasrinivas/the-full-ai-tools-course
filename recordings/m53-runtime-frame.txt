WHY: M-5.3 teaches runtime validation vs static analysis. Visual Studio's

PROBE (temporary test, mirrors labA step 2): constructs a request at
PT-2001 / 97110 / MERIDIAN with confidenceScore = ApprovalService
.AUTO_APPROVE_THRESHOLD (the constant, never the literal), mocks eligibility
to pass, runs process(), and prints the frame.

REAL OUTPUT (verbatim, unedited):

  16:41:37.296 [main] INFO com.priorauth.service.ApprovalService --
      PA request for patientRef=PT-2001 routed to MANUAL_REVIEW (score 0.85 < 0.85)
  === RUNTIME FRAME AT THE THRESHOLD DECISION ===
    score                  = 0.85
    AUTO_APPROVE_THRESHOLD = 0.85
    score >  threshold     = false
    score >= threshold     = true
    Double.compare         = 0
    resulting status       = MANUAL_REVIEW
  ===============================================
  Tests run: 1, Failures: 0 · BUILD SUCCESS

WHAT THIS PROVES (all three are module beats):
 1. PREDICTION CONFIRMED: M-3.4's static trace predicted the routing log would
    print the literally-false "score 0.85 < 0.85". Here it is, emitted by the
    running application, unprompted, in the same output as the frame.
 2. FLOAT HYPOTHESIS KILLED: Double.compare(score, threshold) == 0 — the two
    values are exactly equal. "Maybe it's floating-point drift" is the first
    hypothesis a debugger session exists to test; runtime answers it in one
    line, and the answer is no.
 3. BOTH OPERATORS OBSERVED IN THE SAME FRAME: > is false and >= is true on
    the identical pair of values. The bug is not inference; it is observation.

DOC VERIFICATION (Visual Studio, same day, learn.microsoft.com):
 - Debugger Agent "provides an end-to-end agentic workflow that validates bugs
   against real runtime behavior instead of relying on static analysis."
 - Six documented steps: Context injection · Autonomous reproducer ·
   Hypothesis and instrumentation (tracepoints + conditional breakpoints) ·
   Runtime validation (live telemetry) · Targeted correction ("a precise fix
   at the exact failure point rather than broad refactoring") ·
   **Final human validation** ("You rerun the scenario and confirm the fix in
   the live environment alongside the agent").
