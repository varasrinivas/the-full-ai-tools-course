=== HALF B: BUG-WEB-01, the wasteful re-render ===
Repo: priorauth-web lab/start clone, branch drill/m56-render.
Plant f6f72d8 "Flatten RequestTable to a single component" removed a memoized
Row subcomponent and inlined the rows with fresh inline handlers per render.
MEASURE (temporary render-counting probe, 50-row table):
    === RequestTable render cost (50 rows) ===
      row cells computed on first render      : 50
      row cells recomputed on identical poll  : 50
      wasted recomputes (data unchanged)      : 50   *** every row, every poll ***
  Mechanism: the dashboard polls (usePriorAuthStatus/useReviewQueue re-render the
  parent every 5s); with no memoized Row and inline onClick/onKeyDown closures,
  React re-renders all 50 rows even when the row data is byte-identical.
  Fix direction (taught, profiler-verified in lab): extract a React.memo'd Row +
  a stable onSelect (useCallback) so only changed rows recompute → 0 wasted on
  identical data. Instrument = React DevTools Profiler (no headless equivalent;
  the render-count probe above is the honest CLI stand-in and it is real).
