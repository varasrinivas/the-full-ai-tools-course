THE BUG THE REGENERATED TESTS CAUGHT (a real latent gap in validate()):
  Input confidenceScore = '   ' (whitespace only).
  Code: const score = Number(values.confidenceScore)
  In JS, Number('   ') === 0 (whitespace coerces to 0, NOT NaN). The guard is
    confidenceScore === '' || Number.isNaN(score) || score < 0 || score > 1
  '   ' is not '', 0 is not NaN, 0 is in [0,1] → NO error set. So a
  whitespace-only confidence field is silently ACCEPTED and submit() produces a
  payload with confidenceScore: 0 — a value the user never entered.
  The deleted tests presumably guarded this; regenerating them against the SPEC
  resurrected the catch.
