# Spec: per-payer auto-approve threshold override

## Intent
Some payers negotiate stricter or looser auto-approval. Allow an optional
per-payer threshold that overrides the global auto-approve threshold.

## Acceptance criteria
1. Payer gains an optional `autoApproveThreshold` (nullable decimal).
2. Decision rule: a request auto-approves when confidenceScore >= effective
   threshold (inclusive). Effective threshold = the payer's override when
   present, otherwise the global 0.85.
3. A request scoring exactly the effective threshold auto-approves
   (inclusive boundary).
4. An override outside 0.50–0.99 (inclusive) is rejected at write time.
5. Seed data: at least one payer with an override, and one request scoring
   exactly that override.
6. Tests: boundary at the global 0.85, boundary at a payer override,
   missing override falls back to global.

## Out of scope
Changing the global threshold value; any UI; authentication.
