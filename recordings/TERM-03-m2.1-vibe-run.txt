## What changed

The denials-by-reason widget existed as a bare scaffold (flat red SVG bars, cramped mono labels, no context). I rebuilt it into a proper chart, keeping the no-chart-library approach:

- **`src/components/DenialReasonsChart.tsx`** — rewritten as an HTML/CSS bar list (handles text truncation and hover far better than SVG). Each row shows the reason code as a small mono chip, the human-readable reason text, and a right-aligned count with its share of total (e.g. `9 75%`), above a rounded bar on a quiet track. Bars stay single-hue (one measure = one color, per dataviz practice), sorted descending with an alphabetical tiebreak, with a `title` tooltip per row and a footnote totaling the denials. The `Denials by reason code` aria-label and `data-reason` attributes are preserved.
- **`src/pages/Dashboard.tsx`** — the aggregation now keeps a representative `reasonText` per reason code (the API returned it but the scaffold discarded it), so the chart can show readable labels instead of just codes.
- **`src/index.css`** — new chart styles: gradient bars (`#d0705a → var(--bad)`) on faint tracks, hover brightening, a scale-in animation guarded by `prefers-reduced-motion`, a dashed-border empty state, and a card treatment (panel background, border, radius) for the dashboard sidebar so the widget sits in a proper panel.
- **`src/test/DenialReasonsChart.test.tsx`** — the "widest first" test now queries `[data-reason]` divs and compares percentage widths instead of SVG `rect` attributes, plus a new test covering reason text, share percentages, and the total footnote. The Dashboard test needed no changes.

Nothing is committed — the working tree has the changes ready for you to review and test.
